How It Works

Column Detection
The system automatically finds the phone_number and state columns by header name. If your file has a different header, the system falls back to column positions 7 and 4 (1-indexed).

What Gets Stored
Only phone_number (normalized to 10-digit integer) and state are stored. All other columns (fname, lname, address, city, zip) are ignored.

Performance
Uses PostgreSQL COPY for bulk loading — approximately 1M rows every 30–60 seconds. Large files (250M rows) take 15–25 minutes. Upload runs in the background.

Scrub Jobs
Once data is loaded, user scrub jobs will automatically cross-check against this database. Numbers not found = clean. Numbers found = DNC flagged.
